AMD Kit

A simple, easy to use and understand AMD loader for building more modular sites without using a clunky library.

Asynchronous Module Definitions - Simplified

AMD is a very useful way to organize site Javascript and to speed up page load times. This is a basic JS module loader designed to be so simple that it's easy to extend. Out of the box it doesn't require any specific directory organization or defining include paths beforehand - though organizing your code is good practice!

Easy to use!

Defining a cross-browser console module:

//define the module name (filename requires no setup)
define('/js/utils/console.js', function() {
    //return the module object to be passed around
    if (typeof console === 'object' && typeof console.log === 'function')
        return console.log;
    else
        return alert;
});

anyfile.js / anypage.html: (use the module)

//define dependency modules, pass in as arguments
require('/js/utils/console.js', function(log) {
    //use the arguments to refer to the required module!
    log('AMD Kit worked!');
});

Parallel Script Loading

By loading Javascript files asynchronously you are allowing the page to continue loading while the module handler grabs Javascript dependencies in parallel. Now load your pages faster!

Code Organization

Make it simple to break functionality out into different modules. Without a module loader you might manage your code by adding or changing the HTML script tags (which is a pain itself if working with a compiled backend) or making a huge master file. Now you can break your module into a closure and put it into it's own file.

Enforce Best Practices

If working on a team with less experienced Javascript developers, you can teach this simple version of the module pattern and which can restrict modules to closures and keep the global namespace clean. It also introduces development in self-contained modules rather than a large string of procedural code.

Load When Needed

Rather than loading all of a site's Javascript on every page when needed, it is possible to load only the required Javascript for a page by defining dependencies. You can also wait to load UI extensions one step ahead of user interaction for just in time script loading - especially useful for single page or large applications.
